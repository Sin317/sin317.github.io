Introduction to React: Understanding How It Works and Renders#

What is React?#

React is a popular JavaScript library for building user interfaces, particularly single-page applications where UI updates dynamically without requiring full-page reloads. Developed and maintained by Meta (formerly Facebook), React has become one of the most widely used front-end frameworks due to its declarative and component-based nature.

In this blog post, we’ll dive into how React works, how it renders UI updates efficiently, and what makes it such a powerful tool for modern web development.

Why Use React?#

Before diving into how React works, let’s look at some reasons why developers prefer it:

  • Component-Based Architecture: React applications are built using reusable components, making the code modular and maintainable.
  • Virtual DOM for Efficient Rendering: React updates only the necessary parts of the UI instead of re-rendering the entire page.
  • Unidirectional Data Flow: React enforces a one-way data flow, making it easier to manage application state.
  • Rich Ecosystem and Community Support: React has extensive third-party libraries, tools, and a large community to help developers.
  • Cross-Platform Development: React Native allows developers to build mobile applications using React.

Now, let’s explore how React works under the hood.

How React Works#

Components and JSX#

At its core, React is all about components. A React application is composed of multiple components that represent different parts of the UI. Components can be either functional (simpler and based on functions) or class-based (stateful and based on ES6 classes). However, with the introduction of hooks, functional components have become more prevalent.

React uses JSX (JavaScript XML), a syntax extension that allows developers to write HTML-like code within JavaScript:

function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

This JSX gets compiled into regular JavaScript using Babel before being rendered to the browser.

Virtual DOM and Reconciliation#

React uses a Virtual DOM (VDOM) to optimize UI updates. The Virtual DOM is a lightweight representation of the actual DOM, allowing React to make efficient updates without modifying the real DOM directly.

Here’s how React updates the UI efficiently:

  1. When a component’s state or props change, React creates a new Virtual DOM tree.
  2. It then compares the new Virtual DOM with the previous one using a diffing algorithm.
  3. React determines the minimal set of changes required and updates only those parts of the real DOM (a process called reconciliation).

This approach significantly improves performance because modifying the real DOM is slow, whereas updating the Virtual DOM is much faster.

React Rendering Lifecycle#

React follows a lifecycle for rendering components. Understanding this lifecycle is crucial for managing state updates effectively.

Mounting (Initial Rendering)#

When a component is first rendered, React performs the following steps:

  1. Calls the constructor (for class components) to initialize state.
  2. Calls render() to generate the Virtual DOM.
  3. Updates the real DOM and invokes lifecycle methods like componentDidMount() (for class components) or useEffect(() => {...}, []) (for functional components).

Updating (Re-rendering)#

When a component’s state or props change, React performs an update cycle:

  1. React detects the change and re-renders the Virtual DOM.
  2. It compares the new Virtual DOM with the previous one.
  3. It updates only the changed parts of the real DOM efficiently.
  4. Lifecycle methods like componentDidUpdate() or useEffect() (with dependencies) run after updates.

Unmounting (Cleanup)#

When a component is removed from the DOM, React cleans up any lingering effects using componentWillUnmount() (for class components) or useEffect(() => {...}, []) return function (for functional components).

React State and Props#

State and props are crucial concepts in React:

  • State: A component’s internal data that can change over time (e.g., user input, API responses).
  • Props: Read-only data passed from a parent component to a child component.

Example of using state:

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

Here, useState(0) initializes the count state, and setCount updates it.

React Hooks#

Hooks allow functional components to use state and lifecycle features previously available only in class components. Some important hooks include:

  • useState(): Manages component state.
  • useEffect(): Handles side effects like API calls.
  • useContext(): Accesses global state without prop drilling.
  • useRef(): References DOM elements or persists values across renders.

Example of useEffect() for fetching data:

import React, { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('https://api.example.com/data')
      .then(response => response.json())
      .then(data => setData(data));
  }, []);

  return <div>{data ? JSON.stringify(data) : 'Loading...'}</div>;
}

Handling Events in React#

React events work similarly to native DOM events but use camelCase (e.g., onClick instead of onclick). Event handlers are passed as functions.

Example:

function Button() {
  const handleClick = () => {
    alert('Button clicked!');
  };

  return <button onClick={handleClick}>Click Me</button>;
}

Conditional Rendering#

React allows rendering components conditionally using JavaScript expressions:

function Greeting({ isLoggedIn }) {
  return isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in.</h1>;
}

Lists and Keys#

When rendering lists, React requires a unique key prop to optimize updates.

function List() {
  const items = ['Apple', 'Banana', 'Cherry'];
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item}</li>
      ))}
    </ul>
  );
}

Conclusion#

React’s efficiency comes from its Virtual DOM, reconciliation process, and component-based structure. Understanding how React renders UI updates helps developers write better, more optimized applications.

If you’re new to React, start with small projects and experiment with state, props, and hooks. With practice, you’ll quickly grasp the power of React and be able to build dynamic, scalable applications.
